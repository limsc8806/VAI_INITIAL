# VAI_PLAN 파이프라인 품질 개선안

## 1. 환경/모듈 설치 자동화
- 필수 모듈(Python, PyMuPDF, Camelot 등) 설치 체크 및 자동화 스크립트 제공
- LLM API Key 환경변수 누락 방지: 실행 전 체크 및 안내

## 2. 요구사항/산출물 구조 정규화
- 모든 요구사항/산출물(description, evidence, source_pages 등) JSON 구조 일관성 강화
- 산출물 생성 시 스키마 검증 로직 추가(필수 필드 누락 방지)

## 3. LLM 응답 품질 관리
- 요약 품질/신뢰도 기준 강화, 스텁 요약 최소화
- confidence 값 기준 미달 시 재요약/검증

## 4. 추가 테스트 및 검증
- 각 단계별 자동화 테스트 케이스 추가
- 산출물의 정보 완전성/정확성 검증

---

## 단기 권장 작업
- 환경/모듈 설치 스크립트 개선 및 실행 전 체크
- 요구사항/산출물 스키마 검증 및 정규화 함수 추가
- LLM API Key 관리 및 응답 품질 모니터링

## 중기 권장 작업
- 파이프라인 각 단계별 자동화 테스트/검증 강화
- 요구사항/산출물의 구조적 일관성 및 정보 완전성 지속 개선
- LLM 응답 정규화 및 품질 기준 강화

---

## 개선안 적용 절차
1. 개선안 문서 공유 및 팀 내 리뷰
2. 환경/모듈 자동화 스크립트 배포 및 적용
3. 산출물 생성 코드에 스키마 검증/정규화 로직 추가
4. LLM 응답 품질 관리 및 테스트 케이스 작성
5. 단계별 개선 적용 후 결과 검증 및 추가 피드백 반영

---

## 변경/적용 시 관리 규칙
- 모든 문서/스키마/출력 구조 변경 시 반드시 md파일 및 산출물에 변경 이력/사유 기록
- 변경 전/후 diff 및 영향도 분석 필수
- 변경 내용은 팀 내 공유 및 승인 후 적용
